## 项目背景

Cariad 是大众集团的软件子公司，负责 VW / Audi / Porsche 等品牌的软件平台开发。  
车端数据采集是这个平台的核心基础设施——决定了后续 AI 训练、数据分析、远程诊断、个性化服务等所有上层应用能不能跑通。

**这个项目是"百万级车型平台"的数据采集产品 owner**，从需求分析 → 采集规范 → 质量标准 → 测试验收 → 上线维护，端到端负责。

---

## 关键挑战

1. **多品牌车型平台的数据规范统一**——VW / Audi / Porsche / Cupra 等品牌的基础架构差异
2. **数据合规与跨境传输**——大众集团是欧洲公司，必须同时满足 GDPR、中国《数据安全法》《个人信息保护法》
3. **数据质量 KPI 化**——延时、去包率、覆盖率 是车端数据采集的核心质量指标，直接影响下游所有应用

---

## 我的工作

### 产品设计

- **车端数据采集产品定义**：车辆状态、驾驶行为、故障码 等核心数据
- **采集规范 + 质量标准**：基于平台项目进行需求分析，制定数据字典
- **端到端流程 owner**：从需求 → Spec → 开发 → 测试 → 验收 → 上线

### 跨部门协调

- **品牌售后 / 经销商 / 市场团队**：优化数据采集系统订单，提升数据交付质量
- **数据合规团队**：确保所有数据采集合规，包括**数据跨境传输与用户隐私保护**
- **开发 / 测试 / 平台团队**：协调适配，保障各环境端到端测试通过

### 数据驱动 + 大模型应用

- **基于大模型优化车险方案**（这是我做的最 AI 化的项目之一）
- **JIRA / KPM** 系统处理问题清单 + 进度跟踪
- 数据交付质量（延时、去包等核心 KPI）持续优化

## 复盘与方法论

### 做对的

- **早期介入数据合规**：把合规要求嵌入 PRD 模板，让合规成为产品设计的"前置条件"而非"事后补丁"
- **数据字典统一语言**：跨 4 个时区 + 多个品牌 + 多个 Tier-1，必须有"共同语言"
- **KPI 化数据质量**：把"延时、去包率"作为产品验收硬指标，让数据质量可衡量
